Featured Vulnerabilities

Learn how common web attacks work and how this project mitigates them.

1. SQL Injection (SQLi)

What is it?
An attacker interferes with the queries that an application makes to its database.

Vulnerable:
Uses raw string formatting to construct queries. Passing ' OR 1=1-- in the login completely bypasses the password check.

Secure:
Uses Parameterized Queries (Prepared Statements), treating input strictly as data, not executable SQL commands.

2. Cross-Site Scripting (XSS)

What is it?
An attacker injects malicious executable scripts into the content of a trusted website.

Vulnerable:
User input (like a search query) is reflected directly back to the browser without HTML encoding, causing the browser to execute it.

Secure:
Automatically HTML-escapes all user-provided variables, rendering <script> tags as harmless plain text.

3. IDOR Vulnerability

What is it?
An application provides direct access to objects based on user-supplied input without proper authorization.

Vulnerable:
The backend retrieves a user profile merely by reading the ID from the URL (e.g., /profile/1), allowing anyone to view it.

Secure:
The backend verifies that the requested ID explicitly matches the ID bound to the securely signed JWT session token.

4. Weak Passwords & Brute Force

What is it?
Attackers guess weak passwords or automate login attempts to force their way into accounts.

Vulnerable:
An application accepts "password" or allows 100 failed login attempts in a minute.

Secure:
This app enforces complex password policies upon registration and monitors failed authentications to throw Brute Force alerts!
